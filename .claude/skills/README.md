# Skills

A skill is a repeatable workflow written down so a good output becomes a recipe instead of a lucky prompt.

Each skill is a folder containing a `SKILL.md` with frontmatter (`name`, `description`, optional `argument-hint`) and instructions for when to use it and how. Supporting files (references, examples, templates) can live alongside it in the same folder.

## Build skills organically

Don't write skills in advance for workflows you imagine you'll need. Build a skill the second time you find yourself doing the same thing twice. That way the skill captures what actually worked.
